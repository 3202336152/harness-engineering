# 本地使用指南

这份文档现在只保留为“入口页”，不再继续承载所有细节。

原因很简单：

- “能力介绍”应该单独讲清楚
- “安装、触发、试点操作”也应该单独讲清楚

如果继续混在一个文件里，读者会很难快速找到自己真正关心的内容。

## 现在该看哪一份

| 你的问题 | 建议文档 | 说明 |
|----------|----------|------|
| 这个 Skill 到底提供了什么能力 | [./能力与功能说明.md](./能力与功能说明.md) | 看能力边界、产物结构、治理闭环、适用场景 |
| 我只想看三个核心命令怎么用 | [./命令使用说明.md](./命令使用说明.md) | 看 `/harness init`、`/harness audit`、`/harness plan` 的详细说明 |
| 我怎么在本地安装、远程安装、触发、试点到别的项目 | [./安装与试点指南.md](./安装与试点指南.md) | 看安装方式、命令入口、触发方式、试点步骤 |
| 我只想快速知道仓库入口 | [../README.md](../README.md) | 对外简介和核心命令 |
| 我想看 Skill 契约 | [../SKILL.md](../SKILL.md) | Agent 视角的能力和脚本入口 |
| 我想看方法论 | [./手册/Harness工程手册.md](./手册/Harness工程手册.md) | Harness Engineering 的设计理念 |
