# PR Workflow

> Deep reference for harness-engineering.

## Goal

Make pull requests small, verifiable, and easy for humans to review at the
intent level.

## Required Sections

- What changed
- Why it changed
- Verification performed
- Evidence when relevant

## Review Mindset

- Check whether the change solves the intended problem.
- Check whether it respects architecture and conventions.
- Check whether tests and docs changed with behavior.
- Look for AI-generated complexity that does not earn its cost.

## Good PR Characteristics

- Single responsibility
- Clear validation steps
- Easy rollback path
